第 05 个办公场景

从零造工具：把格式转换器聊出来

想要高效办公，唯有套路得 AI。办公不是拼指令，而是会交代活。 Codex-叉叉 办公实战只记一个套路：问补做。

今天这件活

场景

你经常收到同事发来的 CSV 和 Markdown，要临时转成 JSON、表格或 HTML。在线网站能用，但内部资料传出去不放心。

问补做

问　先说你现在最烦的事。

补　说清材料在哪、要改成什么样、哪些不能碰。

做　聊清楚了，结果就出来了。

把活聊清楚，也就做完了。

一次对话，顺着往下聊： 问→ 补→ 做
问

先说你不想再找在线网站

你不用先说技术方案，只要说清楚反复遇到的麻烦和担心。

我经常要把同事发来的 CSV 和 Markdown 转成别的格式，每次找在线网站都很麻烦，还担心资料传出去。能不能帮我做个本地小工具？

做工具时，第二步要补的是边界：输入是什么、输出是什么、在哪里运行、哪些事情不能发生。

补

把工具形态和边界补齐

这一步说清“本地网页、浏览器里完成、不上传外网”。

做成一个本地网页就行，所有转换都在浏览器里完成，不要上传外网。资料样例在桌面上的“格式转换素材”文件夹，结果放到桌面上的“格式转换工具”文件夹。工具要能把 CSV 转成 JSON 和表格预览，也能把 Markdown 转成 HTML 预览。最好带 README 和示例文件。

工具做好以后，先看它是不是能打开、能预览、能下载结果。不要一开始就追求功能大全。

做

看它交付了什么

这次输出了一个可以直接打开的静态网页工具。

结果里有 index.html、app.js、styles.css、README.md 和 examples；功能包括 CSV 转 JSON、CSV 表格预览、Markdown 转 HTML 预览。
本篇交付物

做到这样，小工具就能先用起来

这个工具不是复杂系统，但已经解决了一个明确的办公痛点。

  • 不需要上传内部资料到外网。
  • CSV 可以转 JSON，也可以直接看表格预览。
  • Markdown 可以转成 HTML 预览，并下载结果。
  • index.html：直接打开使用
  • app.js：转换逻辑
  • styles.css：页面样式
  • README.md：使用说明
收工前，把这个套路带走

做小工具时，先把最常做的那一步做顺

下次你反复做同一个转换动作，不用先想技术栈。先说清楚麻烦，再补输入、输出和边界，Codex-叉叉 就能先做一个能打开、能验证、能继续改的小版本。

问：我反复遇到什么麻烦。 补：输入是什么，输出要什么，哪些边界不能碰。 做：先验证能打开、能转换、能下载。

不是先会写工具，而是先把小工具要替你省哪一步聊清楚。
